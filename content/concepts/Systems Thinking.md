# Systems Thinking

A discipline for understanding how things influence one another within a whole. It treats the world not as a set of linear chains ("x causes y") but as a web of interdependencies with feedback loops.

## Why everyday language fails

Spoken language is linear — "x causes y." This implies one-way causation. In reality, x and y usually affect each other over time, creating circular relationships and feedback loops that linear framing misses entirely.

Systems thinking offers a richer language designed for circularity.

## Key elements of the language

From *Systems Thinking Tools* (p. 6):

- **Closed interdependencies** — focuses on circular, not linear, relationships.
- **Visual language** — loop diagrams (causal loop diagrams) show the system in a snapshot.
- **Precision** — syntactical rules reduce ambiguity and miscommunication.
- **Explicitness of mental models** — forces implicit assumptions into the open.
- **Collective inquiry** — a shared visual model defuses defensiveness; everyone sees the same system, not an us-vs-them debate.
- **Whole over parts** — no isolated blame; everyone is part of the same system.

## Levels of understanding

Systems thinking distinguishes four levels of engagement with any situation (from *Systems Thinking*, p. 9):

| Level | Orientation | Question asked |
|-------|-------------|----------------|
| Events | Reactive | How do I react to this right now? |
| Patterns of events | Proactive | What trends keep recurring? |
| Systemic structure | Creative | What organizational structures create those patterns? |
| Shared vision | Generative | What stated/unstated visions generate those structures? |

Reacting only at the event level is firefighting. Insight lives at the structural and vision levels.

## Core loop types

- [[Reinforcing Loop]] — amplifies change in one direction (virtuous or vicious cycle).
- [[Balancing Loop]] — resists change, pulling the system back toward equilibrium.

Real systems combine both. See each page for examples.
