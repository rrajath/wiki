# Productivity Improvement

Accomplishing a mundane task with **less effort** — through partial or full automation, fewer steps, custom scripts, aliases, or any change that reduces cognitive load or time.

## Characteristics

- Domain-specific: git productivity, CLI productivity, editor productivity.
- Often incremental: a single alias or shortcut.
- Compoundable: individual improvements combine into [[Workflow]]s that are more powerful than the sum of their parts.

## Examples

- A git alias that replaces a long command with two characters.
- A keyboard shortcut replacing a menu click.
- An [[App Summon]] binding that eliminates Cmd+Tab navigation.

## Scale

A single productivity improvement reduces friction on one task. A well-designed [[Workflow]] chains many improvements into a fluid sequence that reduces friction across an entire context switch or work session.
