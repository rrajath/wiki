# Priorities are Time-Bound

Every task has a time component: "higher priority" means it must be done *sooner*, not merely that it matters more in the abstract. Priority and urgency are always linked.

## Core insight

Saying a task is "high priority" is really saying its value decays faster — it must be acted on before the window closes. A task with no deadline is effectively low priority, regardless of how important it sounds.

When you assign priorities, you are implicitly making a claim about time. Making that claim explicit forces clearer thinking: *by when* does this need to be done, and what happens if it slips?

## Connection to scheduling

This is foundational to treating prioritization as a [[Prioritization as a Knapsack Problem|knapsack problem]]: the "value" of each task is not static — it is a function of time. A task due tomorrow has higher value today than a task due next month, even if both are "important."
