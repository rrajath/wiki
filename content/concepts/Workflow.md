# Workflow

A sequence of [[Productivity Improvement]]s combined into a seamless, unified flow of steps. Individual improvements are powerful; combined into a workflow, they multiply.

## Core idea

A single productivity improvement (e.g., a keyboard shortcut to enable Do Not Disturb) is useful in isolation. But chaining it with launching the right app, hiding distractions, and entering a focused state creates a *workflow* — a single gesture that transitions you from scattered to deep work.

## Examples

**Context-switch workflow:**
Enable DND → [[App Summon]] target app → close irrelevant windows → full screen.
Each step alone saves seconds. Together they reliably create a focused context in one motion.

**Git workflow in Emacs:**
Stage files → diff → commit/amend → send code review — all within a single editor window, no terminal switching.

## Why composition matters

Individual [[Productivity Improvement]]s are often domain-specific ("git productivity," "CLI productivity"). When you map them together across the seams between domains, you discover a new class of improvements that wouldn't be visible from inside any single domain.

Many seemingly trivial improvements, seen in context of a workflow, spark ideas about how to handle mundane tasks more efficiently.
