#+title: DynamoDB Data Model
#+filetags: :database:

- [[id:32AE166E-B72D-4D76-B8F4-5EDE5416AFEB][DynamoDB]] is schema-less. It doesn't have to conform to a rigid schema
- Pros:
  - more flexible
  - only pay for what you need
  - add attributes whenever you need them
  - no need to think about migrations
- Cons:
  - it is up to the application to enforce the data consistency
  - the application needs to handle missing attributes
