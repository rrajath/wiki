#+title: Gossip Protocol

Peer to peer scheme for distributing info between nodes. Universal knowledge of the cluster makes every node aware and able to participate in all operations of the database, eliminating any single points of failure and allowing Cassandra to be a very reliable database for [[id:6EE33772-83EE-4051-BDF2-3AAB3F36165A][Availability]]-skewing system designs.

Nodes track various information about the cluster such as what nodes are alive/accessible, what the schema is, etc. They manage the ~generation~ and ~version~ numbers for each node they know about. The ~generation~ is a timestamp when the node was bootstrapped. The ~version~ is a logical clock value that increments every second. Across a cluster, these values form a vector clock. This vector clock allows nodes to ignore old cluster state information when it's received via gossip.
