#+title: Availability in Distributed Systems

Availability is where clients get data they are looking for even if one or more of the nodes go down.

This is where you return a response no matter what - whether the request was successful or not.
This ensures that the system is available.

* How would High Availability influence the design of a distributed system?
- you can use multiple replicas, like multiple read replica
- CDC is by definition strongly consistent
