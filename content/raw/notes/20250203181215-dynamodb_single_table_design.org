#+title: DynamoDB Single Table Design
#+filetags: :database:

This is where all entities share the same table. The partition keys and sort keys are carefully chosen.
* When should one use the single table design?
Mainly applicable in the following scenarios:
- the access patterns are consistent and stable
- no crazy joins
- no transactions that involve more than 10 items
